# 技能目标

通过逐步构建代码，而不是一次性输出完整代码，来降低理解成本，让用户能够阅读、理解实现过程

# 核心规则

- 每次仅实现一个清晰的逻辑单元
- 输出后暂停，等待用户确认或提问
- 必要时简要说明实现思路

逻辑单元示例：

- 数据结构定义
- 核心函数骨架
- 单个功能模块

# 推荐构建流程

按以下顺序逐步实现（可根据实际情况调整）：

1. 整体结构或接口定义
2. 核心数据结构
3. 主流程骨架
4. 功能模块逐步补全
5. 边界与异常处理
6. 优化与重构
